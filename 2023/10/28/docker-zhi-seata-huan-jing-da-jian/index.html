Docker系列-Seata环境搭建

Docker Seata
Docker
发布日期:   2023-10-28
更新日期:   2023-10-28
文章字数:   1.1k

单例环境

1.1、准备工作

（1）创建本地数据卷目录

mkdir -p /home/docker/containers/seata/conf

（2）拉取镜像

docker pull seataio/seata-server

1.2、启动容器

（1）把nacos容器中的文件复制出来

可以先创建一个容器，配置文件复制出来后，再删除这个容器，然后执行第（2）步的操作。

docker cp -a seata:/seata-server/resources/application.example.yml /home/docker/containers/seata/conf

拷贝出来后，将配置文件名修改为application.yml，然后打开该配置文件，根据自己的需求在配置文件中进行自定义修改，修改完成后保存。

（2）启动容器

docker run -d \
--name seata \
-p 8091:8091 \
-p 9091:9091 \
-e SEATA_IP=192.168.0.102 \
-e SEATA_CONFIG_NAME=file:/root/seata-config/registry \
-v /home/docker/containers/seata/conf/application.yml:/root/seata-config \
-v /home/docker/containers/seata/conf/application.yml:/seata-server/resources/application.yml \
--network net-dev \
seataio/seata-server:latest

注意事项：

  • SEATA_IP参数的值为宿主机IP地址。
  • network网络最好与nacos服务在同一网络中。

1.3、配置防火墙

可选配置：配置仅允许指定IP访问seata服务的8091端口：

firewall-cmd --permanent --add-rich-rule='rule family="ipv4" source address="192.168.0.107" port protocol="tcp" port="8091" accept'

# 重新加载，使其生效
firewall-cmd --reload

1.4、docker compose方式搭建

seata服务的docker-compose.yml文件内容参考如下：

version: "2"
services:
  seata:
    image: seataio/seata-server:latest
    container_name: seata
    environment:
    - SEATA_IP=192.168.0.102
    - SEATA_CONFIG_NAME=file:/root/seata-config/registry
    volumes:
    - /home/docker/containers/seata/conf/application.yml:/root/seata-config
    - /home/docker/containers/seata/conf/application.yml:/seata-server/resources/application.yml
    ports:
    - "8091:8091"
    - "9091:9091"
