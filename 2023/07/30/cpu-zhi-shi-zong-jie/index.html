计算机系列-CPU

CPU
计算机
发布日期:   2023-07-30

1、CPU

1.1、认识CPU

1.1.1、CPU简介

CPU 是英文 Central Processing Unit 的首字母缩写，中文意为 中央处理器。

CPU相当于计算机的大脑，其内部 由 数百万 至 数亿 个晶体管 组成。

CPU是各种功能寄存器的集合体。

1.1.2、CPU周期

CPU周期可描述为：首先从内存中取出指令，解码以确定其类型和操作数，接着执行之，然后取指、解码并执行下一条指令。按照该方式，程序被执行完成。

1.1.3、CPU指令集

每个CPU都有一套可执行的 专属指令集。所以，Pentium不能执行SPARC程序，而SPARC也不能执行Pentium程序。

由于用来访问内存以得到指令或数据的时间要比执行指令花费的时间长得多，因此，所有的CPU内都有一些用来保存关键变量和临时数据的寄存器。

1.1.4、CPU流水线

一个CPU可以有分开的取指单元、解码单元和执行单元，当它执行指令n时，它还可以对指令n+1解码，并且读取指令n+2。这样一种机制称为流水线(pipeline)。

对于多数流水线来说，一旦一条指令被 放入 流水线中，它就必须被执行完毕，即便前一条指令是条件转移，它也必须被执行完毕。

比流水线更先进的设计是一种 超标量CPU。

超标量CPU 中有 多个执行单元，两个或更多的指令被同时取出、解码并装入一个保持缓冲区中，直至它们执行完毕。只要有一个执行单元空闲，就检查保持缓冲区中是否还有可处理的指令，如果有，就把指令从缓冲区中移出并执行之。

这种设计存在一种隐含的作用，即程序指令经常不按顺序执行。在多数情况下，硬件负责保证这种运算的结果与顺序执行指令时的结果相同，但是，仍然有部分令人烦恼的复杂情形被强加给操作系统处理

1.1.5、用户态与内核态

多数CPU都有 用户态 和 内核态 两种模式。程序状态字(Program Status Word，PSW)寄存器中有一个 二进制位 控制这两种模式。

用户态 和 内核态 的权限也不同，区别分析如下：

  • 内核态

    当在内核态运行时，CPU可以执行指令集中的每一条指令，并且使用硬件的每种功能。

  • 用户态

    在用户态下运行时，仅允许执行整个指令集的一个子集和访问所有功能的一个子集。

1.2、CPU内部组成

CPU和内存 是由 许多晶体管组成的【电子部件】，通常称为 IC(Integrated Circuit，集成电路)。从功能上来说，CPU内部由【寄存器、控制器、运算器、时钟】四个部分构成，各部分之间通过 电流信号 相互连通。

1.2.1、寄存器

1.2.1.1、寄存器简介

寄存器 中存储的是 指令 和 数据。数据分为 用于运算的数值 和 表示内存地址的数值 两种。

寄存器特点如下：

  1. 不同类型的CPU，其内部寄存器的数量、种类 以及 寄存器存储的数值范围 都不相同。
  2. CPU中每个寄存器的功能都不同。
  3. 数据种类不同，存储该数据的寄存器也不同。

1.2.1.2、寄存器分类

根据功能不同，寄存器可大致分为以下八种：

  1. 累加寄存器(accumulator register)：存储 运算数据 和 运算后数据。
  2. 标志寄存器(flag register)：存储 运算处理后的 CPU状态。
  3. 程序计数器(program counter)：存储 下一条指令所在内存的 地址。
  4. 基址寄存器(base register)：存储 数据内存的 起始地址。
  5. 变址寄存器(index register)：存储 基址寄存器的 相对地址。
  6. 通用寄存器(general purpose register)：存储 任意数据。
  7. 指令寄存器(instruction register)：存储 指令。CPU内部使用，程序员无法干预。
  8. 栈寄存器(stack register)：存储 栈区域的 起始地址。

每一种寄存器允许存在的数量也有限制，分析如下：

  • 允许一个：累加寄存器、标志寄存器、程序计数器、指令寄存器、栈寄存器；
  • 允许多个：基址寄存器、变址寄存器、通用寄存器。

除了以上寄存器外，还有一个 程序状态字（Program Status Word，PSW）寄存器。该寄存器包含了条件码位（由比较指令设置）、CPU优先级、模式（用户态或内核态），以及各种其他控制位。用户程序通常读入整个PSW，但是，只对其中的少量字段写入。在系统调用和I/O中，PSW的作用很重要。

1.2.2、控制器

控制器使命：把内存中的指令、数据等读入寄存器，并根据指令结果来控制整个计算机。

1.2.3、运算器

运算器使命：运算 从内存 读入寄存器 的数据。

1.2.4、时钟

时钟使命：负责 发出 CPU开始计时的 时钟信号。

时钟信号 英文叫做 clock puzzle。Pentium 2GHz 表示 时钟信号的频率为 2GHz（1GHz = 10亿次/秒）。也就是说，时钟信号的频率越高，CPU运行速度越快。

注意：有些计算机的时钟 位于 CPU外部。

1.2.5、总结

寄存器、控制器、运算器和时钟之间的工作方式和关系 可描述如下：

程序启动后，控制器 会根据 时钟信号 从内存中 读取 指令和数据。通过 对这些 指令加以 解释和运行，运算器 会 运算这些数据，控制器 根据 运算结果 来控制计算机。

控制 指的是 数据运算外的处理（主要是 数据输入输出的 时机控制）。

1.3、内存

1.3.1、简介

通常所说的 内存 指的是 计算机主存储器(main memory)，简称 主存。主存 通过 控制芯片 与 CPU 相连，主要负责 存储 指令和数据。

主存 由 可读写的元素 构成，每个字节都带有一个地址编号。CPU可以通过 该地址 读取主存中的指令和数据 和 写入数据。

注意：主存中存储的 指令和数据 会随着计算机的 关机 而 自动清除。

1.4、CPU是寄存器集合体

1.4.1、简介

CPU四个组成部分中，需要重点了解 寄存器，其余三个可不用太过关注。因为 程序是把寄存器 作为对象 来描述的。

1.4.2、汇编与反汇编

汇编语言 采用 助记符(memonic) 来编写程序，每一个原本是 电器信号 的机器语言指令 都会有一个 与其相应的 助记符，助记符通常是 指令功能的英语单词的简写。例如 mov 和 add 分别是数据的 存储(move) 和 相加(addition) 的简写。

汇编语言 和 机器语言 基本上是 一一对应。

  • 汇编：将 汇编语言编写的程序 转化成 机器语言 的过程 称为 汇编。
  • 反汇编：将 机器语言程序 转化成 汇编语言程序 的过程 称为 反汇编。

注意：汇编语言 是 80386以上的CPU 所使用的语言。

汇编示例程序如下所示：

mov eax, dword ptr [ebp-8]     ; 把 数值 从内存 复制到 eax
add eax, dword ptr [ebp-0Ch]   ; exa的数值 和 内存数值 相加
mov dword ptr [ebp-4], eax     ; 把 eax的数值（上一步计算结果）存储在内存中

示例代码解析如下：

  • eax：表示 累加寄存器。
  • ebp：表示 基址寄存器。

1.4.4、条件分支和循环机制

程序的流程分为 顺序执行、条件分支和循环 三种。

  • 顺序执行：按照 地址内容的顺序 执行指令。
  • 条件分支：根据条件 执行 任意地址的 指令。
  • 循环：重复执行同一地址的指令。

条件分支 和 循环 中使用的跳转指令，会参照 当前执行的运算结果 来判断是否跳转。

无论当前 累加寄存器运算结果 是负数、零还是正数，标志寄存器都会将其保存。

CPU运算时，标志寄存器 的数值 会根据 运算结果 自动设定。条件分支 在跳转指令前 会进行 比较运算。是否跳转指令，则由CPU 参考 寄存器数值 后进行判断。运行结果的正、零、负三种状态由标志寄存器的三个位来表示。

  1. 第一字节位为1时，表示 正数。
  2. 第二字节位为1时，表示 零。
  3. 第三字节位为1时，表示 负数。

1.4.5、函数调用

1.4.5.1、简介

函数调用 一般是通过 把程序计数器的值 设定成 函数的存储地址 来实现。

函数调用 需要 在完成函数内部处理后，使得处理流程再返回到函数调用点。

函数的 调用原点 和 被调用函数地址 之间的数据传递，可以通过 内存 或 寄存器 来实现。

1.4.5.2、调用指令

函数调用使用的是 call指令，而不是 跳转指令。

在将 函数入口地址 设定到 程序计数器 之前，call指令 会把 调用函数后要执行的 指令地址 存储在 名为栈的主内存。函数处理完毕后，再通过 函数出口 来执行 return命令。return命令 功能是 把保存在栈中的地址 设定到 程序计数器中。

1.4.6、CPU可执行的指令

按照功能划分，可将CPU能执行的机器语言指令 分为以下几种：

  • 数据转送指令：寄存器和内存、内存和内存、寄存器和外围设备 之间的 数据读写操作。
  • 运算指令：使用 累加寄存器 执行算术运算、逻辑运算、比较运算 和 移位运算。
  • 跳转指令：实现 条件分支、循环、强制跳转。
  • call/return指令：函数调用 / 返回调用前的地址。

CPU可以执行的 指令 非常少。虽然编写的程序看起来非常复杂，但CPU实际处理很简单。
