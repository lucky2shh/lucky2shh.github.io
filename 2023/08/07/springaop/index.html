Spring系列-SpringAOP

Spring SpringAOP
Spring
发布日期:   2023-08-07
更新日期:   2023-08-06
文章字数:   1.9k

基础知识

1.1、使用场景

  • Authentication（权限）
  • Caching（缓存）
  • Context passing（内容传递）
  • Error handling（错误处理）
  • Lazy loading（懒加载）
  • Debugging（调试）
  • logging / tracing / profiling and monitoring（记录跟踪 / 优化 / 校准）
  • Performance optimization（性能优化）
  • Persistence（持久化）
  • Resource pooling（资源池）
  • Synchronization（同步）
  • Transactions（事务）

1.2、AOP思想

AOP思想 的实现技术主要有Spring AOP 和 AspectJ。

AspectJ底层通过静态代理实现编译时自动编译得到一个新类，新类增强了原有类功能。

Spring AOP底层通过动态代理实现运行时增强目标方法，不会生成新类。Spring AOP支持 JDK动态代理 和CGLib代理。

Spring 允许使用 AspectJ Annotation 来定义 方面（Aspect）、切入点（Pointcut）和增强处理（Advice），且可根据这些Annotation来生成AOP代理。Spring 仅仅使用了和AspectJ 5一样的注解，并没有使用AspectJ的编译器或者织入器（Weaver）。

1.3、重要概念

  • 连接点：目标对象中可以被增强的方法；
  • 切入点：目标对象中实际被增强的方法；
  • 通知：增强的代码逻辑；
  • 切面：增强和切入点的组合；
  • 织入：将通知和切入点动态组合的过程；

1.4、切点表达式

1.4.1、语法

切点表达式的语法如下：

execution([访问修饰符] 返回类型 包名.类名.方法名(参数))

注意事项：

  • 访问修饰符可以省略不写；
  • 返回值类型、包名、类名、方法名可以使用*代替，表示任意；
  • 包名与类名之间使用一个点表示该包下的类，两个点表示该包及子包下的类；
  • 参数可以使用两个点表示任意参数；

1.4.2、常用表达式示例

execution(* cn.shh.xxx.A.*(..))：类A下的任意方法都匹配；
execution(* cn.shh.xxx.*.*(..))：包xxx下的任意类的任意方法都匹配；
execution(* cn.shh.xxx..*.*(..))：包xxx及子包下的任意类的任意方法都匹配；
execution(* *..*.*(..))：任意包下任意类的任意方法都匹配；

1.5、通知类型

Spring AOP的通知分如下几种：

  1. 前置通知 Before advice；
  2. 后置通知 After (finally) advice；
  3. 返回/引入通知 After returning advice：引入通知只需要进行配置即可，目的是用来指定哪些方法需要执行相应的通知，如，我们想指定只有 sayHello() 方法执行前置通知；
  4. 异常通知 After throwing advice；
  5. 环绕通知 Around advice；

使用AOP

1.1、实现方式

实现AOP有如下几种方式：

  • 使用xml实现；
    • Advisor方式
    • AspectJ方式
  • 使用注解实现。
  • 使用ProxyFactoryBean实现；
  • 使用ProxyFactory实现；

1.2、XML实现

1.2.1、Advisor方式

xml配置内容如下：

<bean id="test_xml2aop_service" class="cn.shh.demo.spring.aop.impl.xml2aop.Xml2AopServieImpl"/>

<bean id="test_xml2aop_advice" class="cn.shh.demo.spring.aop.impl.xml2aop.Xml2AopAdvice"/>

<aop:config proxy-target-class="true">
    <aop:pointcut id="test_xml2aop_pointcut" expression="execution(* cn.shh.demo.spring.aop.impl.xml2aop.Xml2AopServie.*())"/>
    <aop:advisor advice-ref="test_xml2aop_advice" pointcut-ref="test_xml2aop_pointcut"></aop:advisor>
</aop:config>

Java类文件内容如下：

public class Xml2AopAdvice implements Advice, Ordered {
    @Override
    public int getOrder() {
        return 1;
    }
    void before(){
        System.out.println("--- xml before advice ---");
    }
    void after(){
        System.out.println("--- xml after advice ---");
    }
    void afterReturning(){
        System.out.println("--- xml afterReturn advice ---");
    }
    void afterThrowing(){
        System.out.println("--- xml throwing advice ---");
    }
    Object aroundAdvice(ProceedingJoinPoint pjp) throws Throwable{
        System.out.println("Xml2AopAdvice.aroundAdvice before");
        Object result = pjp.proceed();
        System.out.println("Xml2AopAdvice.aroundAdvice after");
        return result;
    }
}
//====================
public interface Xml2AopServie {
    void info();
}
//====================
public class Xml2AopServieImpl implements Xml2AopServie {
    @Override
    public void info() {
        System.out.println("Xml2AopServieImpl.info");
    }
}

1.2.2、AspectJ方式

xml配置内容如下：

<bean id="test_xml2aop_service2" class="cn.shh.demo.spring.aop.impl.xml2aop.Xml2AopServieImpl"/>

<bean id="test_xml2aop_advice2" class="cn.shh.demo.spring.aop.impl.xml2aop.Xml2AopAdvice2"/>

<aop:config proxy-target-class="true">
    <aop:aspect id="test_xml2aop_aspect" ref="test_xml2aop_advice2">
        <aop:pointcut id="test_xml2aop_pointcut" expression="execution(* cn.shh.demo.spring.aop.impl.xml2aop.Xml2AopServie.*())"/>
        <aop:before pointcut-ref="test_xml2aop_pointcut" method="before" />
        <aop:after pointcut-ref="test_xml2aop_pointcut" method="after" />
        <aop:after-returning pointcut-ref="test_xml2aop_pointcut" method="afterReturning"/>
        <aop:after-throwing pointcut-ref="test_xml2aop_pointcut" method="afterThrowing"/>
        <aop:around pointcut-ref="test_xml2aop_pointcut" method="aroundAdvice" />
    </aop:aspect>
</aop:config>

Java类文件内容与Advisor方式中的一样。

1.3、注解实现

（1）创建增强类AspectJ.java

@Aspect
public class AspectJ {
	//@Pointcut("execution(* doing(..))")
	@Pointcut("@annotation(cn.shh.demo.spring.aop.impl.aspectj2aop.AspectJAnnotation)")
	private void pointcut(){}
	@Before("pointcut()")
	public void before(){
		System.out.println("--- aspectj before advice ---");
	}
}

（2）创建配置类AopConfig.java，开启AOP功能

@Configuration
@EnableAspectJAutoProxy // 开启aop功能
public class AopConfig {
}

1.4、ProxyFactoryBean实现

（1）创建增强通知类

public class PFBAdvice implements MethodInterceptor, Advice {
	@Override
	public Object invoke(MethodInvocation invocation) throws Throwable {
		System.out.println("--- ProxyFactoryBean before advice ---");
		Object proceed = invocation.proceed();
		System.out.println("--- ProxyFactoryBean after advice ---");
		return proceed;
	}
}

（2）使用main方法测试效果

public class App {
	public static void main(String[] args) {
		ProxyFactoryBean factoryBean = new ProxyFactoryBean();
		factoryBean.setTarget(new PFBServiceImpl());
		factoryBean.addAdvice(new PFBAdvice());
		PFBServiceImpl pfbService = (PFBServiceImpl) factoryBean.getObject();
		pfbService.doing();
		pfbService.doing2();
	}
}

1.5、ProxyFactory实现

（1）创建增强通知类

public class PFAdvice implements MethodInterceptor {
	@Override
	public Object invoke(MethodInvocation invocation) throws Throwable {
		System.out.println("--- ProxyFactory before advice ---");
		Object proceed = invocation.proceed();
		System.out.println("--- ProxyFactory after advice ---");
		return proceed;
	}
}

（2）使用main方法测试效果

public class App {
	public static void main(String[] args) {
		ProxyFactory factory = new ProxyFactory(new PFServiceImpl());
		factory.addAdvice(new PFAdvice());
		PFService proxy = (PFService) factory.getProxy();
		proxy.doing();
		proxy.doing2();
	}
}

原理解析

1.1、动态代理

Spring AOP通过JDK动态代理和CGLib动态代理实现增强。

1.1.1、JDK动态代理

JDK动态代理的前提如下：

  • 目标类实现了接口，默认使用该方式；

JDK动态代理会基于接口创建实现类的代理对象，创建出来的代理对象与目标类同属于接口的实现类，与目标类是平级关系。

原理：获取抽象接口信息（利用Java反射技术），并生成一个实现了接口的动态代理类（字节码），再通过反射生成该代理类的实例对象，在调用具体方法时会被拦截，然后执行代理类的方法逻辑。

1.1.2、CGLIB代理

CGLib动态代理的前提如下：

  • 目标类没有实现接口，且没有被final修饰，则默认使用该方式；
  • 若目标类实现了接口，那么可通过设置属性proxy-target-class=true来强制使用CGLib代理；

原理：通过ASM加载被代理类的class文件，通过修改其字节码生成子类，并在子类中采用方法拦截技术，拦截所有父类方法调用，并顺势织入增强逻辑。

1.2、功能实现原理

1.2.1、入口 aop:config

（1）向容器中注入一个Bean增强器，通过Bean增强器对目标对象进行增强

通过标签解析器 ConfigBeanDefinitionParser 向容器中注入 AspectJAwareAdvisorAutoProxyCreator。AspectJAwareAdvisorAutoProxyCreator 就是一个BeanPostProcessor，会在Bean初始化阶段对相关Bean进行增强。

（2）Bean增强器会使用祖先的方法进行增强

AspectJAwareAdvisorAutoProxyCreator 会使用爷爷AbstractAutoProxyCreator的postProcessorAfterInitialization方法对bean增强。

（3）根据目标对象的实际情况，会选择使用CGLib代理或JDK代理来创建代理进行增强。

1.2.2、入口@EnableAspectJAutoProxy

流程与入口aop:config的类似，详情如下：

（1）向容器中注入一个Bean增强器，通过Bean增强器对目标对象进行增强

通过标签解析器 AspectJAutoProxyBeanDefinitionParser 向容器中注入 AnnotationAwareAspectJAutoProxyCreator。AnnotationAwareAspectJAutoProxyCreator 就是一个BeanPostProcessor，会在Bean初始化阶段对其进行增强。

（2）该bean增强器是aop:config导入的增强器的子类

AnnotationAwareAspectJAutoProxyCreator 就是 AspectJAwareAdvisorAutoProxyCreator 的子类。

（3）Bean增强器会使用祖先的方法进行增强

AnnotationAwareAspectJAutoProxyCreator 会使用 AbstractAutoProxyCreator 的postProcessorAfterInitialization方法对bean增强。

（3）根据目标对象的实际情况，会选择使用CGLib代理或JDK代理来创建代理进行增强。
