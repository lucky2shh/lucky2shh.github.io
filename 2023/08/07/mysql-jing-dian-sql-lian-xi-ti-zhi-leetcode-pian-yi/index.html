1.1、(175)组合两个表

编写一个SQL查询来报告 Person 表中每个人的姓、名、城市和州。如果 personId 的地址不在 Address 表中，则报告为空 null 。

Person表:
+----------+----------+-----------+
| personId | lastName | firstName |
+----------+----------+-----------+
| 1        | Wang     | Allen     |
| 2        | Alice    | Bob       |
+----------+----------+-----------+
Address表:
+-----------+----------+---------------+------------+
| addressId | personId | city          | state      |
+-----------+----------+---------------+------------+
| 1         | 2        | New York City | New York   |
| 2         | 3        | Leetcode      | California |
+-----------+----------+---------------+------------+
输出: 
+-----------+----------+---------------+----------+
| firstName | lastName | city          | state    |
+-----------+----------+---------------+----------+
| Allen     | Wang     | Null          | Null     |
| Bob       | Alice    | New York City | New York |
+-----------+----------+---------------+----------+

#解法1：
select
	p.firstName,
	p.lastName,
	a.city,
	a.state
from Person as p
left join Address as a on p.personId = a.personId

1.2、(176)第二高的薪水

查询并返回 Employee 表中第二高的薪水 。如果不存在第二高的薪水，查询应该返回 null(Pandas 则返回 None) 。

Employee 表：
+----+--------+
| id | salary |
+----+--------+
| 1  | 100    |
| 2  | 200    |
| 3  | 300    |
+----+--------+
输出：
+---------------------+
| SecondHighestSalary |
+---------------------+
| 200                 |
+---------------------+
# 解法1：
select
	IF(count(*) = 0, null, 
	IF(o1.s_level = 2, o1.salary, null)) as "SecondHighestSalary"
from (
	select
		e.*,
		DENSE_RANK() over (order by e.salary desc) as "s_level"
	from Employee as e
) as o1
where o1.s_level = 2

# 解法2：
SELECT (
   SELECT DISTINCT salary
   FROM Employee e
   ORDER BY e.salary DESC
   LIMIT 1 OFFSET 1
) AS SecondHighestSalary;

# 解法3：
SELECT IFNULL(( 
     SELECT DISTINCT salary FROM Employee e 
     ORDER BY e.Salary DESC
     LIMIT 1 OFFSET 1
   ), NULL) AS SecondHighestSalary;

1.3、(177)第N高的薪水

查询 Employee 表中第 n 高的工资。如果没有第 n 个最高工资，结果应该为 null 。

Employee table:
+----+--------+
| id | salary |
+----+--------+
| 1  | 100    |
| 2  | 200    |
| 3  | 300    |
+----+--------+
输出: 
+------------------------+
| getNthHighestSalary(2) |
+------------------------+
| 200                    |
+------------------------+

# 解法1：
select IFNULL(
    select e.salary
    from Employee as e
    order by e.salary desc
    limit (N-1) OFFSET 1
    , null) as getNthHighestSalary(N)

1.4、(178)分数排名

查询并对分数进行排序。排名按以下规则计算:

  • 分数应按从高到低排列。
  • 如果两个分数相等，那么两个分数的排名应该相同。
  • 在排名相同的分数后，排名数应该是下一个连续的整数。换句话说，排名之间不应该有空缺的数字。

按 score 降序返回结果表。

Scores 表:
+----+-------+
| id | score |
+----+-------+
| 1  | 3.50  |
| 2  | 3.65  |
| 3  | 4.00  |
| 4  | 3.85  |
| 5  | 4.00  |
| 6  | 3.65  |
+----+-------+
输出: 
+-------+------+
| score | rank |
+-------+------+
| 4.00  | 1    |
| 4.00  | 1    |
| 3.85  | 2    |
| 3.65  | 3    |
| 3.65  | 3    |
| 3.50  | 4    |
+-------+------+

# 解法1：
select
	s.score,
	DENSE_RANK() over (order by s.score desc) as "rank"
from Scores as s

1.5、(180)连续出现的数字

编写一个 SQL 查询，查找所有至少连续出现三次的数字。

Logs 表：
+----+-----+
| Id | Num |
+----+-----+
| 1  | 1   |
| 2  | 1   |
| 3  | 1   |
| 4  | 2   |
| 5  | 1   |
| 6  | 2   |
| 7  | 2   |
+----+-----+
输出：
+-----------------+
| ConsecutiveNums |
+-----------------+
| 1               |
+-----------------+
# 解法1：
set @p:=0;
set @q:=0;
set @i:=0;
select o1.Num as "ConsecutiveNums"
from (
	select
		l.Num,
		@p:=l.Num,
		IF(@p=@q, @i, @i:=@i+1) as "level",
		@q:=@p
	from Logs as l
) as o1
group by o1.level
having COUNT(o1.Id) >= 3

# 解法2：
select DISTINCT
	l1.Num as "ConsecutiveNums"
from Logs l1 
join Logs l2 on l1.Id = l2.Id - 1 and l1.Num = l2.Num
join Logs l3 on l2.Id = l3.Id - 1 and l2.num = l3.num

1.6、(181)超过经理收入的员工

编写一个SQL查询来查找收入比经理高的员工。

Employee 表:
+----+-------+--------+-----------+
| id | name  | salary | managerId |
+----+-------+--------+-----------+
| 1  | Joe   | 70000  | 3         |
| 2  | Henry | 80000  | 4         |
| 3  | Sam   | 60000  | Null      |
| 4  | Max   | 90000  | Null      |
+----+-------+--------+-----------+
输出: 
+----------+
| Employee |
+----------+
| Joe      |
+----------+
# 解释: Joe 是唯一挣得比经理多的雇员。

# 解法1：
select o1.name as "Employee"
from (
	select
		e.*,
		e2.salary as "m_s"
	from Employee as e
	left join Employee as e2 on e.managerId = e2.id
) as o1
where o1.salary > o1.m_s

# 解法2：
SELECT 
   e.name as Employee
FROM Employee e
join Employee e2 on e.`ManagerId` = e2.id 
	AND e.`salary` > e2.`salary`

1.9、(184)部门工资最高的员工

查找出每个部门中薪资最高的员工。

Employee 表:
+----+-------+--------+--------------+
| id | name  | salary | departmentId |
+----+-------+--------+--------------+
| 1  | Joe   | 70000  | 1            |
| 2  | Jim   | 90000  | 1            |
| 3  | Henry | 80000  | 2            |
| 4  | Sam   | 60000  | 2            |
| 5  | Max   | 90000  | 1            |
+----+-------+--------+--------------+
Department 表:
+----+-------+
| id | name  |
+----+-------+
| 1  | IT    |
| 2  | Sales |
+----+-------+
输出：
+------------+----------+--------+
| Department | Employee | Salary |
+------------+----------+--------+
| IT         | Jim      | 90000  |
| Sales      | Henry    | 80000  |
| IT         | Max      | 90000  |
+------------+----------+--------+
# 解释：Max 和 Jim 在 IT 部门的工资都是最高的，Henry 在销售部的工资最高。

# 解法1：
set @p1:=0;
set @p2:=0;
set @q1:=0;
set @q2:=0;
set @i:=0;
select
	d.name as "Department ",
	o3.name as "Employee ",
	o3.salary as "Salary "
from s3_dept as d
join (
	select
		o2.id,
		o2.name,
		o2.salary,
		o2.departmentId
	from (
		select
			o1.id,
			o1.name,
			o1.salary,
			o1.departmentId,
			@p1:=o1.departmentId,
			@p2:=o1.salary,
			IF(@p1=@q1, IF(@p2=@q2, @i, @i:=@i+1), @i:=1) as "level",
			@q1:=@p1,
			@q2:=@p2
		from (
			select e.*
			from s3_emp as e
			order by e.departmentId asc, e.salary desc
		) as o1
	) as o2
	where o2.level = 1
) as o3 on d.id = o3.departmentId

# 解法2：
select
	d.name as "Department",
	o4.name as "Employee",
	o4.salary as "Salary"
from s3_dept as d
join (
	select
		o2.id,
		o2.name,
		o2.salary,
		o2.departmentId
	from (
		select
			e.*,
			DENSE_RANK() over (order by e.departmentId asc, salary desc) as "level"
		from s3_emp as e
	) as o2 
	join  (
		select
			min(o1.level) as "level2"
		from (
			select
				e.*,
				DENSE_RANK() over (order by e.departmentId asc, salary desc) as "level"
			from s3_emp as e
		) as o1
		group by o1.departmentId 
	) as o3 on o2.level = o3.level2
) as o4 on d.id = o4.departmentId
order by o4.id

# 解法3：
select
	d.name as "Department",
	e.name as "Employee ",
	e.salary as "Salary " 
from s3_emp as e
join s3_dept as d on e.departmentId = d.id
where (e.departmentId, e.salary) in (
	select departmentId, MAX(salary)
	from s3_emp
	group by departmentId
)

1.10、(185)部门工资前三高的所有员工

公司的主管们感兴趣的是公司每个部门中谁赚的钱最多。一个部门的 高收入者 是指一个员工的工资在该部门的 不同 工资中 排名前三 。

编写一个SQL查询，找出每个部门中 收入高的员工 。

Employee 表:
+----+-------+--------+--------------+
| id | name  | salary | departmentId |
+----+-------+--------+--------------+
| 1  | Joe   | 85000  | 1            |
| 2  | Henry | 80000  | 2            |
| 3  | Sam   | 60000  | 2            |
| 4  | Max   | 90000  | 1            |
| 5  | Janet | 69000  | 1            |
| 6  | Randy | 85000  | 1            |
| 7  | Will  | 70000  | 1            |
+----+-------+--------+--------------+
Department  表:
+----+-------+
| id | name  |
+----+-------+
| 1  | IT    |
| 2  | Sales |
+----+-------+
输出: 
+------------+----------+--------+
| Department | Employee | Salary |
+------------+----------+--------+
| IT         | Max      | 90000  |
| IT         | Joe      | 85000  |
| IT         | Randy    | 85000  |
| IT         | Will     | 70000  |
| Sales      | Henry    | 80000  |
| Sales      | Sam      | 60000  |
+------------+----------+--------+

# 解法1：
select
	d.name as "Department",
	o3.name as "Employee",
	o3.salary as "Salary"
from s3_dept as d
join (
	select
		o2.name,
		o2.salary,
		o2.departmentId
	from (
		select
			o1.*,
			@c1:=o1.departmentId,
			@c2:=o1.salary,
			IF(@c1=@o1,IF(@c2=@o2, @i, @i:=@i+1), @i:=1) as "level",
			@o1:=@c1,
			@o2:=@c2
		from (
			select
				e.*,
				@c1:=0,
				@c2:=0,
				@o1:=0,
				@o2:=0,
				@i:=0
			from s3_emp as e
			order by e.departmentId asc, e.salary desc
		) as o1
	) as o2
	where o2.level <= 3
	order by o2.id 
) as o3 on d.id = o3.departmentId

# 解法2：
select
	d.name as "Department",
	e1.name as "Employee",
	e1.salary as "Salary"
from s3_dept as d
join s3_emp as e1 on d.id= e1.departmentId
where 3 > (
	select COUNT(DISTINCT e2.salary)
	from s3_emp as e2
	where e1.departmentId = e2.departmentId and e2.salary > e1.salary
)
order by d.id asc

1.12、(197)上升的温度

编写一个 SQL 查询，来查找与之前（昨天的）日期相比温度更高的所有日期的 id 。

Weather 表：
+----+------------+-------------+
| id | recordDate | Temperature |
+----+------------+-------------+
| 1  | 2015-01-01 | 10          |
| 2  | 2015-01-02 | 25          |
| 3  | 2015-01-03 | 20          |
| 4  | 2015-01-04 | 30          |
+----+------------+-------------+
输出：
+----+
| id |
+----+
| 2  |
| 4  |
+----+
# 解释：
2015-01-02 的温度比前一天高（10 -> 25）
2015-01-04 的温度比前一天高（20 -> 30）

# 解法1：
select
	o2.id
from (
	select
		o1.id,
		o1.recordDate,
		o1.Temperature,
		@cd:=o1.recordDate,
		@ct:=o1.Temperature,
		IF(abs(DATEDIFF(@od, @cd)) = 1,11,22) as "tmp",
		IF(abs(DATEDIFF(@od, @cd)) = 1, IF(@ct > @ot, @v:=@v+1, @v:=1), @v:=1) as "level",
		@od:=@cd,
		@ot:=@ct
	from (
		select
			w.*,
			@od:="1000-01-01",
			@cd:="1000-01-01",
			@ot:=0,
			@ct:=0,
			@v:=0
		from s3_weather as w
		order by w.recordDate asc
	) as o1
) as o2
where o2.level = 2

# 解法2：
select w.id
from s3_weather as w
join S3_weather as w2 
	on w.recordDate = DATE_ADD(w2.recordDate, INTERVAL 1 DAY)
		and w.temperature > w2.temperature

1.13、(262)行程和用户

取消率 计算方式：(被司机或乘客取消的非禁止用户生成的订单数量) / (非禁止用户生成的订单总数)。

写一段 SQL 语句查出 "2013-10-01" 至 "2013-10-03" 期间非禁止用户（乘客和司机都必须未被禁止）的取消率。非禁止用户即 banned 为 No 的用户，禁止用户即 banned 为 Yes 的用户。

Trips 表：
+----+-----------+-----------+---------+---------------------+------------+
| id | client_id | driver_id | city_id | status              | request_at |
+----+-----------+-----------+---------+---------------------+------------+
| 1  | 1         | 10        | 1       | completed           | 2013-10-01 |
| 2  | 2         | 11        | 1       | cancelled_by_driver | 2013-10-01 |
| 3  | 3         | 12        | 6       | completed           | 2013-10-01 |
| 4  | 4         | 13        | 6       | cancelled_by_client | 2013-10-01 |
| 5  | 1         | 10        | 1       | completed           | 2013-10-02 |
| 6  | 2         | 11        | 6       | completed           | 2013-10-02 |
| 7  | 3         | 12        | 6       | completed           | 2013-10-02 |
| 8  | 2         | 12        | 12      | completed           | 2013-10-03 |
| 9  | 3         | 10        | 12      | completed           | 2013-10-03 |
| 10 | 4         | 13        | 12      | cancelled_by_driver | 2013-10-03 |
+----+-----------+-----------+---------+---------------------+------------+
Users 表：
+----------+--------+--------+
| users_id | banned | role   |
+----------+--------+--------+
| 1        | No     | client |
| 2        | Yes    | client |
| 3        | No     | client |
| 4        | No     | client |
| 10       | No     | driver |
| 11       | No     | driver |
| 12       | No     | driver |
| 13       | No     | driver |
+----------+--------+--------+
输出：
+------------+-------------------+
| Day        | Cancellation Rate |
+------------+-------------------+
| 2013-10-01 | 0.33              |
| 2013-10-02 | 0.00              |
| 2013-10-03 | 0.50              |
+------------+-------------------+

# 解法1：
select
	t.request_at as "Day",
	ROUND(SUM(IF(t.status = "completed", 0, 1)) / count(t.status), 2) as "Cancellation Rate"
from trips as t
where t.client_id not in(
	select users_id from users where banned = "yes"
)	and t.driver_id not in(
	select users_id from users where banned = "yes"
) and t.request_at BETWEEN "2013-10-01" and "2013-10-03"
group by t.request_at

# 解法2：
select
	t.request_at as "Day",
	ROUND(SUM(IF(t.status = "completed", 0, 1)) / count(t.status), 2) as "`Cancellation Rate`"
from trips as t
left join (
	select users_id from users where banned = "yes"
) as o1 on t.client_id = o1.users_id
left join (
	select users_id from users where banned = "yes"
) as o2 on t.driver_id = o2.users_id
where o1.users_id is null 
	and o2.users_id is null 
	and t.request_at BETWEEN "2013-10-01" and "2013-10-03"
group by t.request_at

1.15、(550)游戏玩法分析4

编写一个 SQL 查询，报告在首次登录的第二天再次登录的玩家的比率，四舍五入到小数点后两位。换句话说，您需要计算从首次登录日期开始至少连续两天登录的玩家的数量，然后除以玩家总数。

# 解法1：
select
	ROUND(AVG(a2.event_date is not null), 2)
from (
	select 
		a.player_id,
		min(a.event_date) as "firstL" 
	from s3_activity as a
	group by a.player_id
) as o1
left join s3_activity as a2 on o1.player_id = a2.player_id and DATEDIFF(a2.event_date, o1.firstL) = 1

# 解法2：
select 
	round(o3.p_count / (select count(DISTINCT a.player_id) from s3_activity as a), 2) as "fraction"
from (
	select 
		count(DISTINCT o2.player_id) as "p_count"
	from (
		select
			o1.*,
			@op:=o1.player_id,
			@cp:=o1.player_id,
			@cd:=o1.event_date,
			IF(@cp=@op || (@t:=0)!=0, IF(abs(DATEDIFF(@od, @cd))=1 || (@t:=1)!=1, IF(@t=0,@v:=@v+1,0), @v:=1), @v:=1) as "level",
			@op:=@cp,
			@od:=@cd
		from (
			select
				a.*,
				@op:=0,
				@cp:=0,
				@od:="1000-01-01",
				@cd:="1000-01-01",
				@v:=0,
				@t:=0
			from s3_activity as a
			order by a.player_id asc, a.event_date asc
		) as o1
	) as o2
	where o2.level = 2
) as o3

1.16、(570)至少有5名直接下属的经理

查询至少有5名直接下属的经理 。

Employee 表:
+-----+-------+------------+-----------+

| 101 | John  | A          | None      |
| 102 | Dan   | A          | 101       |

| 104 | Amy   | A          | 101       |

| 106 | Ron   | B          | 101       |
+-----+-------+------------+-----------+
输出: 
+------+
| name |
+------+
| John |
+------+

# 解法1：
select
	e2.name
from Employee as e2
join (
	select e.managerId
	from Employee as e
	group by e.managerId
	having count(e.id) >= 5
) as o1 on e2.id = o1.managerId

选出所有 bonus < 1000 的员工的 name 及其 bonus。

1.19、(585)2016年的投资

请你编写一个 SQL 查询，报告 2016 年 (tiv_2016) 所有满足下述条件的投保人的投保金额之和：

  • 他在 2015 年的投保额 (tiv_2015) 至少跟一个其他投保人在 2015 年的投保额相同。
  • 他所在的城市必须与其他投保人都不同（也就是说 (lat, lon) 不能跟其他任何一个投保人完全相同）。
    输入：
Insurance 表：
+-----+----------+----------+-----+-----+
| pid | tiv_2015 | tiv_2016 | lat | lon |
+-----+----------+----------+-----+-----+
| 1   | 10       | 5        | 10  | 10  |
| 2   | 20       | 20       | 20  | 20  |
| 3   | 10       | 30       | 20  | 20  |
| 4   | 10       | 40       | 40  | 40  |
+-----+----------+----------+-----+-----+
输出：
+----------+
| tiv_2016 |
+----------+
| 45.00    |
+----------+

解法1：
select SUM(i.tiv_2016) as "tiv_2016"
from s3_insurance as i
join (
	select DISTINCT pid
	from s3_insurance
) as o1 on i.pid = o1.pid
where i.tiv_2015 in (
	select i.tiv_2015
	from s3_insurance as i
	join (
		select DISTINCT pid
		from s3_insurance
	) as o1 on i.pid = o1.pid
	group by i.tiv_2015
	having count(i.tiv_2015) > 1
) and (i.lat, i.lon) in (
	select i2.lat, i2.lon
	from s3_insurance as i2
	join (
		select DISTINCT pid
		from s3_insurance
	) as o1 on i2.pid = o1.pid
	group by i2.lat, i2.lon
	having count(i2.pid) = 1
)

1.20、(586)订单最多的客户

查找下了 最多订单 的客户的 customer_number 。

Orders 表:
+--------------+-----------------+
| order_number | customer_number |
+--------------+-----------------+
| 1            | 1               |
| 2            | 2               |
| 3            | 3               |
| 4            | 3               |
+--------------+-----------------+
输出: 
+-----------------+
| customer_number |
+-----------------+
| 3               |
+-----------------+

# 解法1：
select
	o1.customer_number
from (
	select
		o.customer_number,
		count(o.order_number) as "o_count"
	from orders as o
	group by o.customer_number
	order by o_count desc
) as o1
limit 0, 1

1.21、(595)大的国家

如果一个国家满足下述两个条件之一，则认为该国是大国 ：

  • 面积至少为 300 万平方公里（即，3000000 km2）
  • 人口至少为 2500 万（即 25000000）

编写解决方案找出 大国 的国家名称、人口和面积。

World 表：
+-------------+-----------+---------+------------+--------------+
| name        | continent | area    | population | gdp          |
+-------------+-----------+---------+------------+--------------+
| Afghanistan | Asia      | 652230  | 25500100   | 20343000000  |
| Albania     | Europe    | 28748   | 2831741    | 12960000000  |

1.23、(601)体育馆的人流量

编写一个 SQL 查询以找出每行的人数大于或等于 100 且 id 连续的三行或更多行记录。

Stadium 表:
+------+------------+-----------+
| id   | visit_date | people    |
+------+------------+-----------+
| 1    | 2017-01-01 | 10        |
| 2    | 2017-01-02 | 109       |
| 3    | 2017-01-03 | 150       |
| 4    | 2017-01-04 | 99        |
| 5    | 2017-01-05 | 145       |
| 6    | 2017-01-06 | 1455      |
| 7    | 2017-01-07 | 199       |
| 8    | 2017-01-09 | 188       |
+------+------------+-----------+
输出：
+------+------------+-----------+
| id   | visit_date | people    |
+------+------------+-----------+
| 5    | 2017-01-05 | 145       |
| 6    | 2017-01-06 | 1455      |
| 7    | 2017-01-07 | 199       |
| 8    | 2017-01-09 | 188       |
+------+------------+-----------+

# 解法1：
select
	o3.id,
	o3.visit_date,
	o3.people
from (
	select
		o1.*,
		@c2:=o1.id,
		IF(@c2-@oc2=1, @v2, @v2:=@v2+1) as "level",
		@oc2:=@c2
	from (
		select s.*, @v2:=0
		from s3_stadium as s
		where s.people >= 100
		order by s.id
	) as o1
) as o3 
where o3.level in (
	select
		o2.level
	from (
		select
			o1.*,
			@c:=o1.id,
			IF(@c-@oc=1, @v, @v:=@v+1) as "level",
			@oc:=@c
		from (
			select s.*, @v:=0
			from s3_stadium as s
			where s.people >= 100
			order by s.id
		) as o1
	) as o2
	group by o2.level
	having count(o2.id) >= 3
)

# 解法2：
select DISTINCT s.*
from s3_stadium as s
inner join s3_stadium as s2
inner join s3_stadium as s3
where s.people > 100 and s2.people > 100 and s3.people > 100
	and (
		(s.id - s2.id = 1 and s.id - s3.id = 2 and s2.id - s3.id = 1) or
		(s2.id - s.id = 1 and s2.id - s3.id = 2 and s.id - s3.id = 1) or
		(s3.id - s2.id = 1 and s2.id - s.id = 1 and s3.id - s.id = 2)
	)
order by s.id

# 解法3：
with t1 as (
	select
		s.*,
		s.id - ROW_NUMBER() over(order by s.id) as "rn"
	from s3_stadium as s
	where s.people >= 100
)
select
	id,
	visit_date,
	people
from t1
where rn in (
	select rn from t1 group by rn having count(id) >= 3
)

# 解法4：
with t1 as (
	select
		s.*,
		s.id - ROW_NUMBER() over(order by s.id) as "rn"
	from s3_stadium as s
	where s.people >= 100
)
select
	o1.id,
	o1.visit_date,
	o1.people
from (
	select *, count(rn) over(partition by rn) as "pb" from t1
) as o1
where o1.pb >= 3
order by o1.id

1.24、(602)好友申请2：谁有最多的好友

写一个查询语句，找出拥有最多的好友的人和他拥有的好友数目。

RequestAccepted 表：
+--------------+-------------+-------------+
| requester_id | accepter_id | accept_date |
+--------------+-------------+-------------+
| 1            | 2           | 2016/06/03  |
| 1            | 3           | 2016/06/08  |
| 2            | 3           | 2016/06/08  |
| 3            | 4           | 2016/06/09  |
+--------------+-------------+-------------+
输出：
+----+-----+
| id | num |
+----+-----+
| 3  | 3   |
+----+-----+

# 解法1：
with t1 as (
	select *
	from (
		select 
			ra.requester_id
		from s3_requestaccepted as ra
		union all
		select
			ra2.accepter_id
		from s3_requestaccepted as ra2
	) as alldata
)
select
	requester_id,
	count(requester_id) as "cn"
from t1
group by requester_id
order by cn desc
limit 0, 1

1.25、(607)销售员

查询没有任何与名为 “RED” 的公司相关的订单的所有销售人员的姓名。

SalesPerson 表:
+----------+------+--------+-----------------+------------+
| sales_id | name | salary | commission_rate | hire_date  |
+----------+------+--------+-----------------+------------+
| 1        | John | 100000 | 6               | 4/1/2006   |
| 2        | Amy  | 12000  | 5               | 5/1/2010   |
| 3        | Mark | 65000  | 12              | 12/25/2008 |
| 4        | Pam  | 25000  | 25              | 1/1/2005   |
| 5        | Alex | 5000   | 10              | 2/3/2007   |
+----------+------+--------+-----------------+------------+
Company 表:
+--------+--------+----------+
| com_id | name   | city     |
+--------+--------+----------+
| 1      | RED    | Boston   |
| 2      | ORANGE | New York |
| 3      | YELLOW | Boston   |
| 4      | GREEN  | Austin   |
+--------+--------+----------+
Orders 表:
+----------+------------+--------+----------+--------+
| order_id | order_date | com_id | sales_id | amount |
+----------+------------+--------+----------+--------+
| 1        | 1/1/2014   | 3      | 4        | 10000  |
| 2        | 2/1/2014   | 4      | 5        | 5000   |
| 3        | 3/1/2014   | 1      | 1        | 50000  |
| 4        | 4/1/2014   | 1      | 4        | 25000  |
+----------+------------+--------+----------+--------+
输出：
+------+
| name |
+------+
| Amy  |
| Mark |
| Alex |
+------+

# 解法1：
select sp.name
from SalesPerson as sp
where sp.sales_id not in (
	select o.sales_id
	from orders as o
	where o.com_id in (
		select c.com_id
		from company as c
		where c.name = "RED"
	)
)
