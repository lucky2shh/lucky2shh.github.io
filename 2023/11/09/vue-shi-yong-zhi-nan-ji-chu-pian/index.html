基础工程

搭建步骤

（1）初始化vue项目。通过命令npm init vue@latest可实现创建一个基础工程。

（2）安装依赖。通过命令npm install可安装能使项目运行起来的核心基础依赖。

（3）启动项目，访问服务。通过命令npm run dev可启动运行项目，启动成功后访问指定IP和端口即可实现访问项目。

基础功能

路由

前言

Vue中关于路由的实现方案有多种，这里以Vue Router为例。

使用步骤

（1）安装vue-router。安装命令为npm install vue-router@4。

（2）创建配置路由

在src/router路径下创建index.js文件，文件内容参考如下：

import { createRouter, createWebHistory } from 'vue-router'
import { useUserStore } from '@/stores'

// createRouter 创建路由实例
// 配置 history 模式
// 1. history模式：createWebHistory  地址栏不带 #
// 2. hash模式：createWebHashHistory 地址栏带 #
// console.log(import.meta.env.DEV)

// vite 中的环境变量 import.meta.env.BASE_URL  就是 vite.config.js 中的 base 配置项
const router = createRouter({
  history: createWebHistory(import.meta.env.BASE_URL),
  routes: [
    { path: '/login', component: () => import('@/views/login/LoginPage.vue') }, // 登录页
    {
      path: '/',
      component: () => import('@/views/layout/LayoutContainer.vue'),
      redirect: '/article/manage',
      children: [
        {
          path: '/article/manage',
          component: () => import('@/views/article/ArticleManage.vue')
        },
        {
          path: '/article/channel',
          component: () => import('@/views/article/ArticleChannel.vue')
        },
        {
          path: '/user/profile',
          component: () => import('@/views/user/UserProfile.vue')
        },
        {
          path: '/user/avatar',
          component: () => import('@/views/user/UserAvatar.vue')
        },
        {
          path: '/user/password',
          component: () => import('@/views/user/UserPassword.vue')
        }
      ]
    }
  ]
})

// 登录访问拦截
router.beforeEach((to) => {
  const userStore = useUserStore()
  if (!userStore.token && to.path !== '/login') return '/login'
})

export default router

（3）集成至Vue

在main.js文件中加入路由相关配置，内容参考如下：

import '@/assets/main.scss'
import '@/stores'
import { createApp } from 'vue'
import App from '@/App.vue'
// 引入创建的路由,index.js名字可以省略，前提名字是index.js
import router from '@/router'

const app = createApp(App)
app.use(router)	// 使用路由
app.mount('#app')

（4）使用router标签

在App.vue文件中使用router标签，实现路由功能。参考内容如下：

<script setup>
import zh from 'element-plus/es/locale/lang/zh-cn.mjs'
</script>

<template>
  <div>
    <el-config-provider :locale="zh">
      <router-view></router-view>
    </el-config-provider>
  </div>
</template>

<style lang="scss"></style>

（5）vue页面中使用

在vue页面的script模块中使用路由来实现页面跳转，示例如下：

<script setup>
import { useRouter } from 'vue-router'

const router = useRouter();
// 跳转到请求路径为/的vue页面，这里就是首页。
// 这句代码可以作为登录逻辑处理中登录成功后要执行的代码。
router.push('/');
</script>

Pinia管理库

概述

Pinia是Vue专属状态管理库，允许跨组件或页面共享状态。

Pinia缺点：默认是内存存储，刷新浏览器会丢失数据。对于该问题，可通过使用Persist插件持久化存储来解决。Persist插件使用步骤参考如下：

  1. 通过命令npm install pinia-persistedstate-plugin可安装Persist插件。
  2. 在pinia实例中使用persist。详情可参考pinia使用步骤中的第（2）步。
  3. 定义store时通过持久化参数来设置。

使用步骤

（1）安装Pinia。安装命令为npm install pinia --save

（2）vue实例中使用pinia

想要自定义pinia实例时可以另外创建一个stores/index.js文件来配置，最终导出即可。自定义配置内容参考：

// stores/index.js
import { createPinia } from 'pinia'
import { createPersistedState } from 'pinia-persistedstate=plugin'
const persist = createPersistedState()
const pinia = createPinia()
pinia.use(persist)
export default pinia

在main.js文件中编写如下内容：

import { createApp } from 'vue'
import App from '@/App.vue'
import pinia from './stores'	// 这里导入的就是自定义pinia实例的配置

const app = createApp(App)
app.use(pinia)
app.mount('#app')

（3）pinia使用实例：定义store

// 路径 stores/user.js
import { defineStore } from 'pinia'
import { ref } from 'vue'
import { userGetInfoService } from '@/apis/user'

//用户模块
export const useUserStore = defineStore(
  'big-user',	// 当前store的名字，具备唯一性。
  () => {		// 函数
    // 配置token
    const token = ref('')
    const setToken = (newToken) => {
      token.value = newToken
    }
    const removeToken = ()=>{
        token.value='';
    }

    const userInfo = ref({})
    const getUserInfo = async () => {
      const { data } = await userGetInfoService()
      // console.log(data);
      userInfo.value = data.code === 0 ? data.data : {}
    }
    const setUserInfo = () => {
      userInfo.value = {}
    }

    return {
      token,
      setToken,
      removeToken,
      getUserInfo,
      userInfo,
      setUserInfo
    }
  },
  {
    persist: true,	// 使用pinia persist插件配置持久化存储
  }
)

（4）頁面中使用store

<script setup>
import { useUserStore } from '@/stores/user.js'
const userStore = useUserStore();
const login = async()=>{
    let result = await userLoginService(loginParam.value);
    // 设置token
    userStore.setToken(result.data);
    router.push('/');
}
</script>

工具整合

前言

package.json中的dependencies和devdependencies的解释：

  • dependencies：开发和上线都需要的第三方包，安装时推荐使用参数--save。
  • devdependencies：仅在开发阶段需要的第三方包，安装时推荐使用参数--save-dev。

安装框架或第三方包时通常会使用一些参数，下面对这些参数进行总结和说明：

  • -S：--save的缩写形式，表示将安装的包放在文件package.json里面的dependencies下。
  • -D：--save-dev的缩写形式，表示将安装的第三方包放在文件package.json里面的devdependencies下。

整合Element-Plus

概述

使用步骤

（1）安装Element-Plus

npm install element-plus --save

参数解析：

  • save：将下载的包放在文件package.json里面的dependencies下。

（2）集成

完整引入只需在main.ts文件中加入如下内容：

// main.ts
import { createApp } from 'vue'
import ElementPlus from 'element-plus'
import 'element-plus/dist/index.css'
import App from './App.vue'

const app = createApp(App)

app.use(ElementPlus)
app.mount('#app')

按需自动导入使用步骤如下：

  1. 安装unplugin-vue-components 和 unplugin-auto-import这两款插件。安装命令：npm install -D unplugin-vue-components unplugin-auto-import。

  2. 在vite.config.ts中加入如下内容：

    // vite.config.ts
import { defineConfig } from 'vite'
import AutoImport from 'unplugin-auto-import/vite'
import Components from 'unplugin-vue-components/vite'
import { ElementPlusResolver } from 'unplugin-vue-components/resolvers'

export default defineConfig({
  // ...
  plugins: [
    // ...
    AutoImport({
      resolvers: [ElementPlusResolver()],
    }),
    Components({
      resolvers: [ElementPlusResolver()],
    }),
  ],
})

切换语言包

这里以中文语言包为例。使用中文语言包可实现分页小插件提示信息中文展示。

使用方法很简单，首先在src/main.js文件中导入中文包的js文件，然后在配置vue实例使用element时传递进去即可。内容参考如下：

import { createApp } from 'vue'
import ElementPlus from 'element-plus'
import locale from 'element-plus/dist/locale/zh-cn.js'
import App from './App.vue'
createApp(App).use(ElementPlus, {locale});

整合Axios

官网文档：https://axios-http.com/zh/docs/intro

（1）安装Axios

npm install axios

（2）集成封装

通常情况下，可以基于axios创建一个API请求调用工具类来来使用。下面将创建一个工具类request.js来示范：

import axios from 'axios'
import { useUserStore } from '@/stores'
import { ElMessage } from 'element-plus'
import router from '@/router'

const baseURL = 'http://big-event-vue-api-t.itheima.net'

const instance = axios.create({
  // TODO 1. 基础地址，超时时间
  baseURL,
  timeout: 10000
})

//请求拦截器
instance.interceptors.request.use(
  (config) => {
    // TODO 2. 携带token
    const userStore = useUserStore()
    if (userStore.token) {
      config.headers.Authorization = userStore.token
    }
    return config
  },
  (err) => Promise.reject(err)
)

//响应拦截器
instance.interceptors.response.use(
  (res) => {
    // TODO 3. 处理业务失败
    // TODO 4. 摘取核心响应数据
    if (res.data.code === 0) return res
    ElMessage.error(res.data.message || '服务异常')
    return Promise.reject(res.data)
  },
  (err) => {
    // TODO 5. 处理401错误
    //错误的特殊情况 => 权限不足 或 token过期 => 拦截到登录
    if (err.response?.status === 401) {
      router.push('/login')
    }
    //错误的默认提示
    ElMessage.error(err.response.data.message || '服务异常')
    return Promise.reject(err)
  }
)

export default instance
export { baseURL }

（3）使用示例

js文件：

import request from '@/utils/request'

/**
 * 用户注册接口
 * @param {*} param0
 * @returns
 */
export const userRegisterService = ({ username, password, repassword }) => {
  return request.post('/api/reg', { username, password, repassword })
}

/**
 * 用户登录接口
 * @param {*} param0
 * @returns
 */
export const userLoginService = ({ username, password }) => {
  return request.post('/api/login', { username, password })
}

vue页面文件：

<script setup>
import { ref, watch } from 'vue'
import { useRouter } from 'vue-router'
import { useUserStore } from '@/stores'
import { userRegisterService, userLoginService } from '@/apis/user'

// 模型数据定义
const userStore = useUserStore()
const isRegister = ref(false)
const refObj = ref(null)
const formModel = ref({
  username: '',
  password: '',
  repassword: '',
  rememberme: false
})

// 注册
const registerFn = async () => {
  // console.log("register");
  await refObj.value.validate()
  const { data } = await userRegisterService(formModel.value)
  // console.log(res);
  if (data.code === 0) {
    ElMessage.success(data.message)
    isRegister.value = false
    clearFormModel()
  } else {
    ElMessage.error(data.message)
  }
}

// 登录
const router = useRouter()
const loginFn = async () => {
  await refObj.value.validate()
  const { data } = await userLoginService(formModel.value)
  if (data.code === 0) {
    ElMessage.success(data.message)
    // console.log(data);
    userStore.setToken(data.token)
    clearFormModel()
    router.push('/')
  } else {
    ElMessage.error(data.message)
  }
}

const clearFormModel = () => {
  formModel.value.username = ''
  formModel.value.password = ''
  formModel.value.repassword = ''
  formModel.value.rememberme = false
}

watch(isRegister, () => {
  clearFormModel()
})
</script>
<template></template>
<style lang="scss" scoped></style>

import { defineConfig } from 'vite';

整合js-cookie（cookie管理）

概述

相关文档及教程：点我直达。

使用步骤

（1）安装js-cookie。安装命令npm install js-cookie --save。

（2）使用

import cookie from 'js-cookie'
// 存储cookie，对当前页面路径有效
cookie.set('name', 'value', { expires: 7, path: '' });

cookie.remove('name', {path: ''});

常见问题

跨域问题

定义

当一个请求url的协议、域名、端口三者之间任意一个与当前页面url不同时即为跨域。

解决方法

解决该问题的方法有多种，这里以vue通过代理解决跨域为例来说明。

在vite.config.js文件中配置如下代码可实现通过代理解决跨域：

import { fileURLToPath, URL } from 'node:url'
import {defineConfig} from 'vite'
import vue from '@/vitejs/plugin-vue'

// https://vitejs.dev/config/
export default defineConfig({
  plugins: {
    vue(),  
  },
  resolve: {
    alias: {
      '@': fileURLToPath(new URL('./src', import.meta.url))
    }
  },
  server:{
      proxy:{
          '/api': {
              target: 'http://localhost:8080',
              changeOrigin: true,
              rewrite: (path) => path.replace('/^\/api/','')
          }
      }
  }
})
